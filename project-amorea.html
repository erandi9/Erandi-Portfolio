Amorea

Amorea is a responsive web application designed to help couples manage wedding invitations and guest lists in a simple, organized, and stress free way.

While the platform allows users to create and customize digital invitations, its core value lies in guest management, enabling couples to track RSVPs, monitor attendance in real time, and gain clear visibility into their event planning.

The experience was designed to work seamlessly across both mobile and desktop, ensuring accessibility and usability regardless of device.

View Final Product

Problem

Planning a wedding is often overwhelming, especially when managing guest lists and RSVP responses.
Couples face several challenges:

  • Lack of visibility into who has confirmed attendance
  • Difficulty tracking responses across multiple channels (messages, calls, emails)
  • No centralized system to monitor guest status
  • Existing tools are either too complex or not designed with usability in mind

This results in disorganized workflows, increased stress, and inefficient planning.

My Role

UX/UI Designer responsible for the end to end product design.

  • Led user research, persona creation, and journey mapping
  • Defined information architecture and navigation structure
  • Designed wireframes and high fidelity prototypes in Figma
  • Built a scalable design system aligned with development needs
  • Collaborated closely with the developer to ensure feasibility and efficient implementation
  • Designed the visual identity, including branding, color system, and typography
  • Currently developing the landing page using HTML and CSS

Approach

The design approach focused on simplifying a complex and stressful process into a clear, intuitive, and user-friendly experience.

  • Conducted user research to understand behaviors, frustrations, and expectations during wedding planning
  • Created personas and journey maps to identify key pain points and opportunities
  • Designed a site map early in the process to define structure and navigation before moving into UI
  • Prioritized clarity, ease of use, and real time visibility across all features

Key Decisions

  • Focused the product on guest management as the core value, rather than invitation creation
  • Designed real time RSVP tracking to give users immediate visibility into attendance
  • Created a dashboard style experience to surface key insights (confirmed, declined, pending)
  • Ensured the platform was fully responsive, optimizing the experience for both mobile and desktop
  • Built a scalable design system with reusable, responsive components to support future growth
  • Designed components with development in mind (React friendly), improving implementation efficiency

System & Scalability

To support long term scalability, a comprehensive component library was developed early in the process.

  • Created a reusable and consistent design system with no duplicated components
  • Ensured all components were responsive and adaptable across devices
  • Structured the system to align with React development, simplifying implementation
  • Reduced future design and development time by standardizing patterns and interactions

Solution

Amorea provides couples with a centralized platform to:

  • Create and send personalized digital invitations
  • Manage guest lists in a structured and organized way
  • Track RSVPs in real time
  • Visualize attendance through clear and actionable insights
  • Reduce manual work and uncertainty during wedding planning

The result is a streamlined, intuitive experience that reduces stress and improves planning efficiency.

Impact

  • Simplified guest management by centralizing all RSVP data in one platform
  • Reduced uncertainty by providing real time visibility into attendance status
  • Improved usability through a clear and structured interface
  • Created a scalable foundation for future product expansion

Reflection

This project highlighted the importance of focusing on the core user problem rather than surface-level features.
While digital invitations are valuable, the real need lies in simplifying guest management and reducing stress during a complex life event.
Designing Amorea required balancing usability, emotional context, and technical scalability to create a meaningful and practical solution.
